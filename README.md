# Banking Assistant RAG Chatbot

A Flask-based banking assistant that connects SQL Server transaction and customer records with FAISS vector search and Google Gemini. The app provides a conversational query interface for banking data, real-time transaction insertion, and a background scheduler to keep the vector index synchronized with the database.

## Project Overview

This repository contains a prototype banking AI assistant that:

- Uses SQL Server (`pyodbc`) with a `Transactions` table and an expected `Customers` table.
- Builds a FAISS vector index from transaction and customer text embeddings.
- Uses Google Gemini embeddings (`gemini-embedding-001`) to vectorize records and retrieve them for RAG queries.
- Serves a web UI using Flask with chat, status dashboard, and transaction insertion.
- Runs a background scheduler to detect database changes and rebuild the FAISS index automatically.

## Key Files

- `app.py` - Flask web server with endpoints for chat, status, and transaction insertion.
- `index_service.py` - Database access, text formatting, embeddings generation, FAISS indexing, and search.
- `scheduler.py` - Background scheduler that polls the database and refreshes the FAISS index.
- `init_db.py` - SQL Server initialization script to create `BankingDB` and populate the `Transactions` table with sample data.
- `requirements.txt` - Python dependencies.
- `templates/index.html` - Front-end UI layout.
- `static/app.js` - Front-end interaction logic.
- `static/style.css` - UI styling.
- `scheduler_status.json` - Scheduler status cache file.
- `index.faiss`, `index.pkl` - Serialized index and metadata data files generated by the app.

## Requirements

- Python 3.11+ (recommended)
- SQL Server accessible from the host
- ODBC driver for SQL Server (`ODBC Driver 17/18` or `SQL Server`)
- Google Gemini API key
- Required Python packages from `requirements.txt`

## Environment Variables

Create a `.env` file in the project root with these values:

```env
DB_SERVER=your_sql_server_host
DB_USER=sa
DB_PASSWORD=your_password
DB_NAME=BankingDB
TABLE_NAME=Transactions
GEMINI_API_KEY=your_gemini_api_key
```

> Note: The repository currently expects a `Customers` table during index building, but `init_db.py` only creates and populates the `Transactions` table. If you want full customer indexing, add a `Customers` table and records before running the app.

## Setup

1. Install dependencies:

```bash
pip install -r requirements.txt
```

2. Configure `.env` with your SQL Server and Gemini credentials.

3. Initialize the database:

```bash
python init_db.py
```

4. Start the app:

```bash
python app.py
```

5. Open the web UI at:

```text
http://localhost:5000/
```

## How It Works

### Web UI

The web UI allows you to:

- Ask natural language banking questions.
- Receive AI responses that reference transaction or customer records.
- Insert new transaction records into the SQL Server database.
- Monitor sync status and indexed record counts.

### Chat Endpoint

`POST /api/chat`

- Accepts JSON with `message`.
- Searches the FAISS index for related records.
- Builds a prompt containing database context and system instructions.
- Calls Gemini model `gemini-2.5-flash` to generate a response.
- Returns the response plus source metadata for display.

### Status Endpoint

`GET /api/status`

- Returns scheduler state, last checked time, last sync time, total DB records, and indexed count.

### Add Transaction Endpoint

`POST /api/add_transaction`

- Accepts JSON transaction fields.
- Validates required fields and date format.
- Inserts a new transaction row into `Transactions`.
- Newly inserted records are indexed on the next scheduler run.

## Notes and Caveats

- The background scheduler runs every 10 seconds by default.
- `index_service.py` saves the FAISS index to `index.faiss` and metadata to `index.pkl`.
- If the index files are missing, the app rebuilds the index automatically.
- The `GEMINI_API_KEY` must be valid for the Gemini SDK calls to work.
- If your database does not contain a `Customers` table, either create it or modify `index_service.py` to only index transactions.

## Recommended Improvements

- Add a `Customers` creation / seed step to `init_db.py`.
- Implement proper exception logging and UI error handling.
- Use stronger type and API validation for user input.
- Add unit tests for `index_service.py` and `scheduler.py`.

## Running Locally

1. Ensure SQL Server is reachable and credentials are correct.
2. Run `init_db.py` once to seed sample data.
3. Run `app.py`.
4. Use the web UI to query and insert transactions.
